# 048 — PWA for `/reservar` Salesperson Form

**Date:** 2026-03-11
**Author:** Jorge Luis Contreras Herrera

## Description

Converts the `/reservar` salesperson form into a Progressive Web App (PWA). Salespeople can now install the form to their home screen, use it like a native app, and submit reservations even without connectivity — queued submissions replay automatically when signal returns.

## What changed

### New: Web App Manifest (`public/manifest.json`)
- App name: "Reservar — Puerta Abierta"
- Standalone display, portrait orientation
- Scoped to `/reservar`
- Theme color `#2563eb` (primary blue), background `#f6f8fb`
- Icons: 192x192 and 512x512 PNG (maskable)

### New: Service Worker (`public/sw.js`)
- **Install:** Pre-caches the `/reservar` shell
- **Navigation:** Network-first with cache fallback (form loads offline)
- **Static assets:** Stale-while-revalidate for `/_next/`, icons, CSS, JS
- **API calls:** Skipped by SW — the offline queue in `use-offline-queue.ts` handles POST failures with localStorage persistence and exponential backoff retry
- **Activate:** Cleans old caches on version bump

### New: PWA Icons (`public/icons/`)
- `icon-192.png` — 192x192, resized from existing `favicon.png`
- `icon-512.png` — 512x512, resized from existing `favicon.png`

### Modified: Root Layout (`src/app/layout.tsx`)
- Added `viewport` export: `width=device-width`, `initial-scale=1`, `viewport-fit=cover`, `theme-color=#2563eb`
- Added to `metadata`: `manifest: "/manifest.json"`, `appleWebApp` (capable, default status bar, title "Reservar"), `icons.apple`

### Modified: Reservar Client (`src/app/reservar/reservar-client.tsx`)
- Registers service worker on mount (`navigator.serviceWorker.register("/sw.js")`)
- Wires `useOfflineQueue` hook into the form flow
- Shows offline indicator banner with pending queue count (`"Sin conexion · N pendientes"`)
- Passes `enqueue` function to `StepReviewSubmit`

### Modified: Review & Submit Step (`src/app/reservar/step-review-submit.tsx`)
- Uses `enqueue()` instead of direct `fetch()` for submission
- When offline, `enqueue()` returns `null` → treated as success ("queued for later")
- When online, `enqueue()` sends immediately and returns the `Response`

## Architecture decisions

| Decision | Rationale |
|---|---|
| No `next-pwa` / `@serwist/next` | Zero dependencies — a 70-line hand-rolled SW is simpler and sufficient for this scope |
| SW skips `/api/` routes | The offline queue (`use-offline-queue.ts`) already handles POST failures with localStorage + exponential backoff — no need for SW to intercept API calls |
| Network-first for navigation | Ensures the latest HTML when online, falls back to cached shell when offline |
| Stale-while-revalidate for assets | Fast loads from cache, background refresh keeps assets current |
| `scope: "/reservar"` | PWA only covers the salesperson form — the rest of the app (admin, dashboard) remains a regular web app |
